const uint32_t  nodes 
)

How the crs should be reshaped for color optimization?

Rebasing the crs based on the returned sequence should reduce cache-misses resulting from color based access to the matrix.

Don't forget to free() the returned sequence when not needed anymore.

Parameters
colorsThe color groups of the nodes.
nodesThe number of nodes in the matrix.
Returns
The optimized index based sequence of nodes.

Creates a new CRS structure to store a matrix.

Should be freed using the NTW_CRS_free when non needed anymore.

Parameters
nodeNumThe number of nodes (rows) of the matrix.
edgeNumThe number of non-zero elements of the matrix (edges).
rowPtrArray of nodeNum + 1 elements with the indeces of the val where each row begins and the end of the val array (where last row ends).
colIndArray of edgeNum elements with the columns of each non zero element of the matrix.
valArray of edgeNum elements with the non zero values of the matrix.
Returns
ntw_crs* The created struct or 0 (NULL) if unsuccessful.
